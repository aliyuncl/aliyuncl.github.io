if语句：if eles if + 条件（bool），如果为真，执行code体，如果为假，执行else code
switch case语句：
1、普通型
var i = 0
switch i { //表达式，case进行一一对比，某个case为真，即可调出switch，需要使用c语言中的break
case 0:
“0” //被输出
case 1:
“1”
default:
“2”
}
2、如果想要执行2条case，那么可以在case中，分割
var i = 3
switch i {
case 0,3:
“0” //被输出
case 1:
“1”
default:
“2”
}
3、如果想要执行完一条case，继续执行下一条case，那么需要使用fallthrough
var i = 0
switch i {
case 0:
“0” //被输出
fallthrough //
case 1:
“1” //被输出
case 2:
“2”
default:
“default”
}
特殊用法：switch支持区间分割
let age = 5
switch age {
case 0…11:
print(“正太”)
case 12…30:
print(“少年”)
default:
print(“大叔”)
}

let dic = {“lily”:19,”baby”:2}
for (aname,iage) in dic{

}

实例变量instance variable和本地变量local variable的区别：
1、本地变量仅仅存在于method的里面，一旦method结束，那么变量就消失了。原理：每次调用方法的时候，都会生成新的变量，老的变量会丢失。
2、实例变量存在于这个object里面，可以在method之外的，一般把他们放在最上面，都可以用。
如果已经声明了实例变量，再在method里面声明本地变量（变量名相同），本地变量只能在method里面是合法的。
注意：一般情况下，建议在实例变量前面在一个_，比如_score，说明是个实例变量，以区分本地变量

极其重要：：：
let alertAction = UIAlertAction(title:”ok”,style: .default,handler: nil)
handler：nil
在实例1中，目前：点击button以后，会执行present—各个method，实际app就是在未点击alertcontroller上面的ok，就已经生成了startroud等，为了解决这个问题，引入闭包的概念：
handler告诉alertController要发生什么，目前是nil，什么都不告诉，继续执行下面的method
如果改成：
handler: { action in
self.startNewRound()
self.updateLabel()
})
这个代码块就叫做闭包，可以理解为一个no name的method，这块代码不会被马上执行，而是等到tap ok才会被执行。
注意：
闭包连，向view controller发送消息，必须使用self，在外部正常代码可以不写，这个swift得规定！

为了向某个object传递message，可以采用以下格式：
reveiver.methodName(parameters),其中receiver就是那个接受消息的object，如果是自己的话，可以用self，一般可以省略
