docker-基础操作

发表于 2017-06-28

基本概念：
底层技术的实现：Cgroup和NameSpace，CPU、内存、IO的限制
内存：
-m指定内存限额
–memory-swap，限制内存+swap的限额，如果在创建时不指定–memory-swap，默认 memory：swap=1：1
CPU:
-c，–cpu-shares，限制cpu的使用权重，默认值是1024，注意只有在cpu资源紧张的情况下，才会按照这个比例来分配
IO:
–blkio-weight，设置的是

网络：none、bridge、host,使用的是–network=host…来使用
none:无网络
bridge：网桥模式
brctl show 可以看到当前host的网桥，可以看到当前网桥上挂载的虚拟网卡有哪些，默认情况下，都是挂载到docker0上面了。但注意网桥内看到的虚拟网卡名称和容器内部看到的不一样的，因为是通过veth pair的方式连接的，可以理解一根网线的两端。同时容器被被分配一个与网桥在一个子网下的IP地址（DHCP），能否指定IP呢？

docker也可以自己建立网桥，docker network create –driver bridge,如何指定吗？
答：可以，docker network create –driver bridge –subnet III –gateway III mynet

指定网桥来创建虚机
docker run -ti –network=mynet,也可以指定IP –ip=
注意：！！只有指定了subnet的网桥，才可以分配静态IP

host：与主机公用网络的模式，但注意host使用过的端口docker就不能再使用了。

镜像相关：
1、docker images，查询当前有多少镜像
bogon:blog alieric$ docker images
REPOSITORY TAG IMAGE ID CREATED SIZE
nginx latest 958a7ae9e569 4 weeks ago 109 MB
bitnami/php-fpm latest 86885793ee60 6 weeks ago 216 MB

2、docker rmi imagesID，删除掉当前某个镜像

容器相关
1、docker ps 正在运行的容器
2、docker ps -a，所有容器
bogon:blog alieric$ docker ps -a
CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
cf89077c66ed nginx “/bin/bash” 53 seconds ago Exited (0) 10 seconds ago fervent_euler
7f5a6f6ded1b nginx “/bin/bash” About a minute ago Exited (0) About a minute ago gifted_thompson
0371d1fe39b0 nginx “nginx -g ‘daemon …” 2 weeks ago Exited (0) 2 weeks ago agitated_heyrovsky
fa996abdaff2 nginx “–name mynginx” 2 weeks ago Created 0.0.0.0:80->80/tcp frosty_knuth
59aba678060a nginx “/bin/bash” 2 weeks ago Exited (0) 2 weeks ago ecstatic_goldberg

created：已创建但未运行的容器
UP：正在运行
exited：运行但已经退出的容器
3、docker rm containerID，删除当前某个容器
4、docker rm $(docker ps -a)，删除所有，类比，docker rmi $(docker images)
